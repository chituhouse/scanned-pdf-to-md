核心方法论

为什么需要双源交叉验证？单一OCR引擎无法解决所有问题

问题：单一OCR的局限性

识别误差

选项错位、文字识别错误、表格结构破损，单一引擎无法完全避免

水印干扰

培训机构水印混入正文（如"小象教育"、"抖音"等），污染内容

格式混乱

选项格式不统一（A.xxx vs A、xxx），括号类型混用

内容缺失

部分选项或题目未被识别，单源无法补齐

解决：双源互补策略

通用OCR

优势：文字完整度高、题号识别稳定
劣势：表格结构差、选项可能连成一行

智能文档解析

优势：表格识别优秀、选项换行准确
劣势：可能漏识部分文字

交叉验证

核心洞察：两个OCR源的错误通常不会出现在同一位置，取长补短即可

四大核心原则

项目成功的关键设计决策

01

双源互补，不是二选一

错误做法：选择"更好"的版本
正确做法：合并两个版本，生成完整内容。一版缺选项D，另一版有，合并即可。

02

逐套处理，不批量操作

教训：批量处理20套题，后期精度明显下降。改为逐套处理，每套确认后再继续，质量可控。

03

铁律原则，防止AI发挥

不发挥：只用源文档已有内容
不编造：缺失标注[缺失]
不修改：只修格式，不改文字

04

验证报告，全程透明

每套题处理后生成报告：修复了什么、来源哪个版本、待人工确认项。可追溯、可复核。

Skill 文件结构要素

部分 作用 示例
Frontmatter 定义名称和描述 name: cross-validate-ocr
输入定义 明确输入文件/数据 纯OCR版.md + 文档解析版.md
工作流程 分步骤描述操作 分割 → 验证 → 格式化 → 输出
验证规则 约束AI行为的铁律 不发挥、不编造、不修改
输出格式 定义输出结构 验证报告 + 正文内容

设计要点

  • 明确输入输出，不留模糊空间
  • 分步骤描述，每步有明确动作
  • 设置铁律约束AI"发挥"倾向
  • 优先级规则处理冲突场景
  • 输出模板确保格式一致

避坑指南

  • 不要批量处理大量数据（精度下降）
  • 不要让AI自行判断"更好"（容易乱改）
  • 不要省略验证报告（无法追溯）
  • 不要用模糊词汇（"适当"、"合理"）
  • 不要跳过人工确认环节

提示词设计要点

Skill 中的关键提示词模式

铁律约束模式

## 验证规则（必须严格遵守） ### 铁律 1. **不发挥、不编造**：只使用两个MD文档中已有的内容 2. **不猜测答案**：如果两版都没有某内容，标注 `[缺失]` 而不是编造 3. **保留原文**：修复格式问题，但不修改题目文字本身

优先级规则模式

### 优先级规则 | 内容类型 | 优先采用 | 原因 | |:---|:---|:---| | 表格 | 文档解析版 | Markdown格式更好 | | 选项文本 | 文档解析版 | 换行更准确 | | 题目文本 | 对比取更完整的 | 两版各有优劣 | | 题号 | 纯OCR版 | 识别更稳定 |

分步执行模式

### 第二步：逐套交叉验证 **重要：必须一套一套处理，绝对不能批量处理！每套处理完必须展示报告给用户确认！** #### 2.1 逐题合并 对每道题执行： 1. 从纯OCR版提取题目内容 2. 从文档解析版提取同一题内容 3. **合并生成完整题目**： - 题目文本：取更完整的一版 - 选项A/B/C/D：对比两版，取有内容的 4. 输出标准格式

冲突处理模式

### 冲突处理 当两版内容明显不一致时： - 保留两者内容 - 添加标注：`[待人工确认：版本A为xxx，版本B为yyy]` - 记录到验证报告中

提示词设计原则

A

具体 > 抽象

❌ "处理格式问题"
✅ "选项格式统一为 A. xxx（字母+点+空格+内容）"

B

规则 > 判断

❌ "选择更好的版本"
✅ "表格优先采用文档解析版"

C

约束 > 自由

❌ "适当补充内容"
✅ "不编造，缺失标注[缺失]"

D

示例 > 描述

❌ "输出标准格式"
✅ 提供完整的输出模板示例

延伸开发指南

基于本方法论开发你自己的 Skill

复刻步骤

1. 识别你的"双源"

任何需要交叉验证的场景都可以套用。例如：两个翻译引擎、两个代码生成模型、两个数据来源。关键是找到互补的两个源。

2. 定义合并规则

明确每种内容类型的优先级。不要让AI自行判断"哪个更好"，而是给出明确规则。

3. 设置铁律约束

AI倾向于"发挥"和"优化"。必须用铁律约束：不编造、不猜测、不过度修改。

4. 设计验证报告

每次处理后输出报告，记录做了什么、来源是什么。这是质量保证的关键。

5. 分批处理

不要一次性处理大量数据。分批处理，每批确认后再继续。

可延伸场景

翻译交叉验证

结合 Google Translate + DeepL，对比合并，提升翻译质量

代码生成验证

结合 GPT-4 + Claude 生成的代码，取各自优势部分

数据清洗验证

结合多个数据源，交叉验证数据准确性

发票OCR验证

结合多个OCR引擎，确保金额、日期等关键字段准确
